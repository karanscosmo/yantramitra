Industrial operations intelligence for a five-plant Indian manufacturing company.

YantraMitra unifies plants, machines, sensors, work orders, digital twins, and AI agents into a single operating layer so teams can understand issues and act quickly.

Scenario

The seeded company operates Pune automotive components, Ahmedabad textile and chemical process lines, Chennai electronics assembly, Bengaluru precision engineering, and a Nagpur logistics hub. Each plant has plant-specific machines, risks, and operational workflows.

Pune Automotive

Robotic welding, CNC cells, AGV movement, and drivetrain component production.

Ahmedabad Process

Textile finishing, dye tanks, process parameters, and chemical quality context.

Chennai Electronics

SMT machines, AOI inspection, reflow ovens, and electronics test benches.

Bengaluru Precision

Micro-machining, metrology, CMM inspection, calibration, and additive manufacturing.

Nagpur Logistics

ASRS aisles, sortation, dock operations, AGVs, and warehouse flow monitoring.

Company-ready notes

Deployment and operating model

Authentication and roles

Public pages explain the platform. Application pages require authentication. Signup supports role selection, and the app separates manager, maintenance, operator, and executive workflows.

Data model

The seeded database contains plants, machines, sensors, alerts, work orders, agent missions, and profile data. The public home page uses the same counts so the landing experience feels connected to the actual product.

AI assistant

YantraNklan can be connected to OpenAI with a company-owned API key. For production, configure secrets in environment variables, restrict access by role, and keep human approval for physical maintenance actions.

Release checklist

  • Use a production JWT secret and environment-managed API keys.
  • Connect the database to company-approved infrastructure.
  • Validate agent recommendations through plant operating procedures.
  • Confirm sensor integrations and work-order ownership rules before live use.
  • Keep public documentation accurate as plant data or workflows change.
